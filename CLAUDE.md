### Project-specific guardrails for any agent

1. **Money is integer rupiah.** Never use floats. Use the helpers in
   `packages/shared/src/money.ts` and the fee model in `fees.ts`. PPN (VAT) is
   11% on the service fee only.
2. **Webhooks must be idempotent and signature-verified.** Midtrans, GoSend, and
   Grab handlers all write to the webhook log table first; never mutate order
   state from an unverified payload.
3. **API is the only service with secrets.** Clients (web + mobile) must never
   import the Supabase service-role key, Midtrans server key, or any courier
   credentials. Those live behind `apps/api`.
4. **RLS is the source of truth for client reads.** When adding a table, add an
   RLS policy in the same migration. Do not rely on client filtering.
5. **Order state transitions go through the API state machine** in
   `apps/api/src/routes` â€” do not let the client write order status directly.
6. **Shared code is the contract.** Domain types and Zod schemas live in
   `packages/shared/src`. If web, mobile, and api each redefine a type, fix the
   shared package instead.
7. **Indonesia-specific:** phone numbers are the primary identifier; encrypt PII
   at rest per PDP Law; scraping adapters should be swappable for official
   partner APIs later.
8. **Don't invent a "First Amended" anything** â€” that's a firm rule from the
   global CLAUDE.md, but it also means: don't rename modules, don't restructure
   folders, and don't "clean up" the workspace layout unless asked.
